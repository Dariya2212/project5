# Новый виджет

## Описание:

Новый виджет - это обновление для личного кабинета клиента банка. Это виджет, который показывает несколько последних успешных банковских операций клиента.

## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/Dariya2212/project5.git
```
2. Установите зависимости:
```
poetry install
```

## Тестирование

1. Необходимо установить зависимости:
```
poetry add --group dev pytest-cov
```
2. Запуск тестов:
```
pytest
```
3. Запуск тестов с оценкой покрытия:
```
poetry run pytest --cov
```
4. Дополнительно можно создать HTML файл для просмотра отчета о покрытии:
```
pytest --cov=tests --cov-report=html
```


## Использование
1. В модуле masks.py написаны следующие функции:
    - get_mask_card_number - принимает на вход номер карты и возвращает ее маску
    - get_mask_account - принимает на вход номер счета и возвращает его маску
   
    
Примеры использования и тестирование каждой из этих функций можно найти в модуле test_masks.py

2. В модуле widget.py написаны следующие функции:
    - mask_account_card - принимает тип и номер карты или счета и возвращает строку с замаскированным номером
    - get_date - Функция принимает на вход строку с датой в длинном формате и возвращает короткий вариант даты

Примеры использования и тестирование каждой из этих функций можно найти в модуле test_widget.py

3. В модуле processing.py написаны следующие функции:
    - filter_by_state - принимает на вход список банковских операций и возвращает отфильтрованный по статусу список
    - sort_by_date - возвращает новый список, отсортированный по дате
    - process_bank_search - принимает список словарей с данными о банковских операциях и строку поиска,
      а возвращает список словарей, у которых в описании есть данная строка
    - process_bank_operations - принимает список словарей с данными о банковских операциях и список категорий операций,
      а возвращает словарь, в котором ключи — это названия категорий,
      а значения — это количество операций в каждой категории

Примеры использования и тестирование каждой из этих функций можно найти в модуле test_processing.py

4. В модуле generators.py написаны следующие функции:
    - filter_by_currency - принимает на вход список транзакций и возвращает итератор, поочередно выдающий транзакции
    - transaction_descriptions - принимает список транзакций и возвращает описание каждой операции
    - card_number_generator - Генератор выдает номера банковских карт в формате XXXX XXXX XXXX XXXX

Примеры использования и тестирование каждой из этих функций можно найти в модуле test_generators.py

5. В модуле decorators.py добавлен декоратор log, который автоматически логирует результаты функции или возникшие ошибки
   Примеры использования:
   - Записывает результат в лог-файл
     ```
     @log(filename="mylog.txt")
     def my_function(x, y):
         return x + y
     ```
   - Выводит результат в консоль, так как лог-файл не указан
     ```
     @log()
     def my_function(x, y):
         return x + y
     ```

6. В модуле external_api.py написаны следующие функции:
    - get_exchange_rate - получает текущий курс обмена для валюты currency к рублю. Возвращает курс как float
    - convert_to_rub - конвертирует сумму из валюты EUR или USD в рубли.

Примеры использования и тестирование каждой из этих функций можно найти в модуле test_external_api.py

7. В модуле financial_operations.py написаны следующие функции:
    - read_financial_operations_csv - функция, считывающая данные о финансовых операциях из csv-файла, 
      и возвращающая список словарей
    - read_financial_operations_excel - Функция, считывающая данные о финансовых операциях из excel-файла,
      и возвращающая список словарей

Примеры использования и тестирование каждой из этих функций можно найти в модуле test_financial_operations.py

8. Добавлен модуль main.py, предоставляет пользовательский интерфейс и связывает все функциональности между собой
